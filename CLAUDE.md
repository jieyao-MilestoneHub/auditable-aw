# AAW - Auditable AI Workspace

Enterprise-grade auditable development framework for **multi-role AI collaboration**.

## Core Concept

AAW lets multiple Claude Code sessions work on the same codebase **without conflicts**. Each session has a defined role with specific boundaries — what it can read, write, and must avoid.

```
Terminal 1          Terminal 2          Terminal 3
    │                   │                   │
    ▼                   ▼                   ▼
/role frontend      /role backend       /role integrator
    │                   │                   │
    ▼                   ▼                   ▼
┌─────────┐         ┌─────────┐         ┌─────────┐
│ Can only│         │ Can only│         │ Can only│
│ touch   │         │ touch   │         │ touch   │
│ frontend│         │ backend │         │ contracts│
└─────────┘         └─────────┘         └─────────┘
```

## Quick Start

### 1. Initialize Project

```bash
claude
/init-project fullstack-ts   # Or: fullstack-py, nextjs, microservices, two-person
```

This creates `.aaw-project.json` with your project structure and role definitions.

### 2. Start Collaboration

Open multiple terminals, each activates a role:

```bash
# Terminal 1
claude
/role frontend TICKET-123

# Terminal 2
claude
/role backend TICKET-123

# Terminal 3 (optional)
claude
/role integrator TICKET-123
```

### 3. Check Status

```bash
/sync-status   # See locks, pending requests, notifications
```

## Configuration: `.aaw-project.json`

This file defines **everything** — roles, paths, boundaries. Auto-generated by `/init-project`.

```json
{
  "name": "my-project",
  "roles": {
    "frontend": {
      "path": "workspace/frontend",
      "framework": "react",
      "canRead": ["workspace/shared/**"],
      "canWrite": ["workspace/frontend/**"],
      "forbidden": ["workspace/backend/src/services/**"]
    },
    "backend": {
      "path": "workspace/backend",
      "framework": "fastify",
      "canRead": ["workspace/shared/**"],
      "canWrite": ["workspace/backend/**"],
      "forbidden": ["workspace/frontend/src/components/**"]
    }
  },
  "shared": {
    "types": "workspace/shared/types",
    "contracts": "workspace/contracts"
  }
}
```

**All project code lives in `workspace/`.** AAW framework stays separate.

## Available Presets

| Preset | Roles | Structure |
|--------|-------|-----------|
| `fullstack-ts` | frontend, backend, integrator | React + Node.js |
| `fullstack-py` | frontend, backend, integrator | React + FastAPI |
| `nextjs` | frontend, backend, integrator | Next.js App Router |
| `microservices` | gateway, user-service, order-service, integrator | Multiple services |
| `two-person` | frontend, backend | Simple 2-role setup |

Or run `/init-project custom` to define your own.

## Governance

This workspace enforces:

- **Permission Control**: Sensitive files are protected
- **Tool Management**: Only approved MCP servers allowed
- **Audit Trail**: All actions are logged
- **Evidence Generation**: Compliance-ready bundles

## Available Commands

| Command | Description |
|---------|-------------|
| `/init-project [preset]` | Initialize project with a preset or custom config |
| `/role <name> [ticket]` | Activate a role (reads from `.aaw-project.json`) |
| `/sync-status` | Check locks, requests, notifications |
| `/aaw-plan` | Start a planning session |
| `/aaw-implement` | Execute implementation |
| `/aaw-test` | Run tests |
| `/aaw-evidence` | Generate evidence bundle |

## Security Rules

- Never read `.env` or credential files
- Never run `sudo` or dangerous commands
- All shell commands go through aawctl
- External connections require approval
